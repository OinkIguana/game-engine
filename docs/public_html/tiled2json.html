tiled2json

The tiled2json command line utility provides a method of changing tilemaps generated by the Tiled editor into the tilemaps used by this engine. Note that this is a one way translation, so if you expect to be editing this map it would be good to save the original Tiled files as well. Running the tiled2json utility does not modify them at all.

In order for the tilemaps to be converted successfully, a few rules must be followed when using Tiled:

  • Orientation is orthagonal
  • Tile layer format is CSV
  • Tile render order is Right Down
  • Layer names are interpreted as the depths, so must be numeric
  • Collisions are an exception – the collision map should be on a layer titled "collisions"
  • The collision map layer interprets any tile as a wall (1), and no tile as a space (0)
  • Tileset names are interpreted as texture page names, so they must be named the same as your texture pages

Usage

node tiled2json.js infile outfile
infile
the path to the Tiled tilemap file (.tmx)
outfile
the path to where the tilemap JSON should be created. If the file already exists, it is overwritten.

Example

node tiled2json.js tilemap.tmx tilemap.json
